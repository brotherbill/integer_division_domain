# 00.00.00.00 — ABBREVIATIONS  
## Canonical Dictionary of Expanded Terms and Abbreviations

## Purpose
This document defines **every abbreviation** used anywhere in the Prax Junior domain.  
Each abbreviation is listed with its **expanded form**, **definition**, and **usage domain**.

Per Prax Junior doctrine:

> **Every abbreviation must be expanded at first use within each `.md` file, followed immediately by the abbreviation in parentheses. After this introduction, the abbreviation may appear naked anywhere else in the same file. Readers must never be forced to consult this dictionary, but this dictionary must remain complete, authoritative, and zero‑drift.**

All entries are alphabetized.

---

# A

### **Abstract Syntax Tree (AST)**
A tree representation of program structure used in compilers and analyzers.  
**Domain:** Language tooling, Prax IDE.

### **Automated Test Equipment (ATE)**
Hardware/software systems used to validate embedded or safety‑critical devices.  
**Domain:** Certification, hardware integration.

---

# B

### **Business Continuity (BC — enterprise)**
The discipline of ensuring systems remain operational during disruptions, outages, or ND‑world failures.  
**Domain:** Enterprise systems, ND world.

### **Bounded Context (BC — DDD)**
A linguistic and conceptual boundary in Domain‑Driven Design where terms have precise meaning and cannot drift.  
**Domain:** Domain‑Driven Design, modeling, D world boundaries.

---

# C

### **Continuous Integration (CI)**
Automated build/test pipelines triggered by code changes.  
**Domain:** Tooling, ND world.

### **Continuous Deployment (CD)**
Automated release pipelines that push changes into production.  
**Domain:** Tooling, ND world.

---

# D

### **Deterministic (D)**
A sealed‑domain execution model where functions behave identically for identical inputs, with predictable timing and no external dependencies.  
**Domain:** Prax Junior D world.

### **Design by Contract (DbC)**
A correctness discipline where functions declare preconditions, postconditions, and invariants.  
**Domain:** Prax Junior, Eiffel heritage.

---

# E

### **Execution Time Budget (ETB)**
The maximum allowed execution time for a deterministic function.  
**Domain:** D world, timing guarantees.

---

# F

### **Failure Mode and Effects Analysis (FMEA)**
A structured method for identifying and mitigating failure modes.  
**Domain:** Safety‑critical engineering.

---

# G

### **Garbage Collection (GC)**
A nondeterministic memory‑management mechanism that introduces unpredictable pauses and global interference.  
**Domain:** ND world only; forbidden in D world.

---

# I

### **Integrated Development Environment (IDE)**
A software environment providing editing, building, debugging, and analysis tools.  
**Domain:** Prax IDE.

### **Instruction Set Architecture (ISA)**
The machine‑level instruction model for CPUs.  
**Domain:** Deterministic timing, POP manual.

---

# M

### **Memory Safety (MS)**
Guarantees preventing invalid memory access, corruption, or leaks.  
**Domain:** `@safe`, D world.

### **Model‑Based Design (MBD)**
A methodology using formal models to generate certified code.  
**Domain:** Safety‑critical systems.

### **MISRA C (MISRA)**
A restricted subset of C used in automotive and safety‑critical systems.  
**Domain:** Certification.

---

# N

### **Non‑Deterministic (ND)**
The world of infrastructure, external dependencies, timing variability, concurrency, and unpredictable behavior.  
**Domain:** ND world.

### **Normalized Data Structure (NDS)**
A cleaned, sealed‑domain representation of ND inputs prepared for deterministic execution.  
**Domain:** ND → D pipeline.

---

# P

### **Prax Junior (PJ)**
The certifiable subset of the D programming language enforcing `@safe pure nothrow @nogc` for deterministic business‑rule execution.  
**Domain:** Entire system.

### **Point of Presence (POP) Manual**
The hardware‑level timing and instruction reference used to compute deterministic execution time.  
**Domain:** D world timing guarantees.

---

# R

### **Real‑Time Operating System (RTOS)**
An OS providing deterministic scheduling guarantees.  
**Domain:** Safety‑critical systems.

---

# S

### **Single Source of Truth (SSOT)**
A discipline ensuring that each piece of data has exactly one authoritative representation.  
**Domain:** ND world, normalization.

### **Safety‑Critical System (SCS)**
Systems where failure can cause harm, requiring deterministic behavior and certification.  
**Domain:** Certification.

---

# T

### **Three‑Letter Acronym (TLA)**
Any abbreviation consisting of three letters.  
**Domain:** Documentation discipline.

### **Worst‑Case Execution Time (WCET)**
The maximum possible execution time of a deterministic function, computed from the POP manual.  
**Domain:** D world timing guarantees.

---

# U

### **Undefined Behavior (UB)**
Any operation whose result is unpredictable or unbounded.  
**Domain:** Forbidden in D world.

---

# Z

### **Zero‑Drift Documentation (ZDD)**
Documentation that cannot accumulate inconsistencies, ambiguity, or divergence from code.  
**Domain:** Prax Junior doctrine.
